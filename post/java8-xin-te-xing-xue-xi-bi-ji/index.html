JAVA8新特性学习笔记

发布于 2020-01-06 10分钟 | 2331字数

对于Java8的一些特性，早有耳闻，趁考完研这几天空闲时间，赶紧把它学了一遍。特别是lambda和stream的新特性，超级棒很 nice，似乎让Java代码更加优雅了。

JAVA8

简介

速度快

代码少，新增了新的语法lambda表达式

强大的Stream API

便于并行扩展，支持

最大化减少空指针异常(optional)

lambda表达式(拉姆达)

箭头操作符

  • lambda拆分两部分

    • 左侧

      • 表达式的参数列表

    • 右侧

      • 表达式执行的功能

语法格式

  • 无参数 无返回值

    • () -> System.out.println("")

  • 有1个参数 无返回值

    • (x) -> System.out:prinln

      • 只有一个参数小括号可以不写

  • 有多个参数 有返回值 lambda语句中有多条语句

    • (x,y) -> { xxxxx;xxxxx;xxxx; return xxx;}

      • 若lambda中只有一条语句时，则打括号和return都可以不写

  • lambda的参数列表的数据类型可以不写，

    • JVM编译器可以通过上下文自动类型推断

  • 总结：左右遇一括号省略，左侧推断类型省略，lambda中一条语句return省。能省则省

lambda表达式需要函数式接口的支持

  • 函数式接口：接口中只有一个抽象方法的接口。
  • 可以使用注解@FunctionalInterface. 当使用注解时，表明该类为函数式接口，类中只能有一个函数

四大核心函数式接口

Consumer(消费型接口)

  • void accept()

Supplier(供给型接口)

  • T get()

Function(T , R )函数型接口

  • R apply(T t)

Predicate 断言型接口

  • boolean test(T t)

方法引用和构造器引用

方法引用

  • 若Lambda体中的内容 已经被实现了，我们可以使用“方法引用”
  • 对象：：实例方法名

Employee employeess = new Employee();
Supplier sup= employeess::getUsername;
employeeess就是一个实例对象，并且调用实例的方法

  • 类：：静态方法名

    • Comparator comparator = (x, y) -> Integer.compare(x, y); Comparator comparator1 = Integer::compare;
      compare为Integer中的静态方法

  • 类：：实例方法名

    • 规则：第一个参数是实例方法的调用者，第二个参数是实例方法的形式参数
      BiPredicate<String, String> biPredicate = (x,y)->x.equals(y);
      BiPredicate<String, String> biPredicate1 = String::equals;

  • 注：lambda体中方法的参数变量要与引用函数的参数变量的类型个数相同，且返回类型相同

构造器引用

  • Supplier supplier = () -> new Employee();
    Supplier supplier1=Employee::new;
    构造器的参数列表取决于函数式接口(@functionInterface)中抽象方法的参数列表

数组引用

  • Function<Integer, String[]> fun = x -> new String[10];
    Function<Integer,String[]> function=String[]::new;

强大的Stream API

简介

  • 是数据渠道，用于操作数据源(集合数组等)所产生的元素序列。
    “集合讲的是数据流讲究的是计算！”
  • 与io流相区分开：stream API在流传输过程中可以进行一系列加工操作后产生一个新的流，数据源并不会发生变化，
  • ⚠️：Stream 自己不会存储数据
    Stream 不会改变源对象。相反，会返回一个持有结果的新的Stream
    Stream 操作是延迟执行的，意味着会等到需要结果时才执行

操作

  • 创建Stream

    • 方式一：通过Collection系列集合提供的Stream()或parallelStream()可以获取流（第一个是串行流，第二个是并行流）
      List list = new ArrayList<>();
      Stream stringStream=list.stream();
    • 方式二：通过Arrays中的静态方法stream()读取数组流
      Employee[] employees = new Employee[];
      Stream stream=Arrays.stream(employees);
    • 方式三：通过Stream类中的静态方法Of()
      Stream streams=Stream.of("aa,","bb","cc");
    • 方式四：创建无限流
      1.迭代
      Stream stream4=Stream.iterate(0,x->x+2 );
      stream4.forEach(System.out::println);
      2.生成

  • 中间操作(不会执行任何操作)

    • 惰性求值

      • 多个中间操作可以连接起来形成一个流水线，除非流水线上触发终止操作，否则中间操作不会执行任何的处理，而在终止操作时一次性全部处理

    • 内部迭代： 迭代操作由Stream API完成
      外部迭代：自己写迭代器

    • 筛选与切片

      • filter

        • 接受lambda，从流中排除某些元素

      • limit

        • 截断流，使元素不超过给定数量

      • skip(n)

        • 跳过元素，返回一个扔掉前n个元素的流。若流中元素不足n个，则返回一个空流。与limit(x)互补

      • distinct

        • 筛选，通过流所生成的hashcode()和equals()去除重复元素

    • 映射

      • map

        • 接收lambda，将元素转换成其他形式或提取信息。接收一个函数作为参数，该函数会被应用到每个元素上，并将其映射成一个新的元素
          List stringList3=Arrays.asList("aaa","bbbb","fadfasd");
          stringList3.stream().map(str->str.toUpperCase()).forEach(System.out::println);

      • flatMap

        • 接收一个函数作为参数，将流中的每个值都替换成一个流，然后把所有流连接成一个流

    • 排序

      • sorted()

        • 自然排序

      • sorted(Comparator)

        • 定制排序

  • 终止操作(一次性执行全部操作)

    • 查找与匹配

      • allMatch

        • 检查是匹配所有元素

      • anyMatch

        • 检查是否匹配至少一个元素

      • noneMatch

        • 检查是否没有匹配所有元素

      • findFirst

        • 返回第一个元素

      • findAny

        • 返回当前流中的任意元素

      • count

        • 返回流中的元素的总个数

      • max

        • 返回流中的的最大值

      • min

        • 返回流中的最小值

    • 归约

      • reduce(T identity, BinaryOperator)

        • 可以将流中的元素反复结合起来，得到一个值
          List integerList=Arrays.asList(1,2,3,4,5,6,7);
          Integer sum=integerList.stream().reduce(0,(x,y)->x+y);//sum
        • map 和reduce 的链接通常称为map-reduce模式
          大数据中应用广泛（map做提取操作，reduce做规约）

      • collection

        • 将流转换为其他形式，接受一个Collection接口实现，用于Stream中元素做汇总的方法
          employees.stream().map(Employee::getUsername)
          .collect(Collectors.toCollection(HashSet::new)).forEach(System.out::println);
        • 多级分组 Collectors.groupingBy
        • 分区：Collectors.partitioningBy()

并行流与顺序流

fork/join框架 ----效率高(传统的多线程发生阻塞后备队列就此阻塞,而fork/join模式采用工作窃取模式，自己没有作业可执行时，窃取其他线程上的任务执行)

将一个大任务进行拆分成若干个小任务(拆到不可再拆时),再将一个个的小任务运算的结果进行join汇总。

parallel() 并行流

  • 速度超快，cpu利用超级高

sequential() 方法

  • 顺序流

Optional类

简介：

是一个容器，代表一个值存在或不存在，原来用null表示一个值不存在，现在Optional可以更好的表达这个概念，可以避免空指针异常

Optional

  • 可以快速定位出问题的位置，1.8之前null报错不能直接报出位置，只能找出通过debug找到

Optional.of(T t)

  • 创建一个optional实例

Optional.empty()

  • 创建一个空的optional

Optional.ofNullable(T t)

  • 若t不为空时，创建optional实例，否者创建实例

isPresent()

  • 判断是否包含值

orElse(T t)

  • 如果调用对象中有值，返回该值，否则返回t

orElseGet(Supplier s)

  • 如果调用对象包含值，返回该值，否则返回s获取的值

map(Function f)

  • 如果有值对其处理，并返回处理后的optional，否则返回optional.empty()

flatMap(Function mapper)

  • 与map类似，要求返回值必须是optional

接口中的默认方法和静态方法

默认方法

  • 1.8以前 只有两种：一种是全局静态常量，一种是抽象方法
  • 新：default方法 默认方法
    default String getName(){}
  • 接口默认方法的“类优先原则”
    若一个接口中定义了一个默认方法，而另一个父类或接口中又定义了一个同名的方法时
    1.选择父类中的方法，如果一个父类提供了具体的实现，那么接口中具有相同名称和参数的方法会被忽略
    2.接口冲突。如果一个父接口提供一个默认的方法，而另个一个接口也提供了一个具有相同名称和参数列表的方法(不管方法是否是默认方法)，那么必须覆盖该方法来解决冲突----------(单继承接口可以多个实现)

静态方法

新时间日期API

旧的时间日期API ------ 不好用，是可变的，都不是线程安全的。多个线程操作，导致时间变化。

如：simpleDateFormat ----可变的，存在现在安全问题
不可变是指：如String定义的这种常量，不能变化，除非其他线程自己创建一个新的实例

java.time.* -------这个包中的方法不可变，线程安全

instant ------- 时间戳
localDate ------本地日期
localTime ------本地时间
localDateTime -----本地日期时间

java.time.chrono -------特殊格式的日期包

java.time.format ------格式化

如：DateTimeFormatter dtfsss = DateTimeFormatter.ofPattern("yyyy年MM月dd日");

java.time.tempral -------时间计算

TemporalAdjuster 时间矫正器
如将日期调到下一个星期等：
LocalDateTime localDateTimes =ldr.with(TemporalAdjusters.next(DayOfWeek.MONDAY));
